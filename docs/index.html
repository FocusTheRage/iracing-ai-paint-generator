UV-aware generation

Downloads official iRacing PSD templates, extracts paintable masks and wireframes, and guides AI output to valid paint islands.

Regional paint control

Prompt for specific panels — hood, doors, rear quarter, and more — on cars with regional atlas data.

Install-ready output

Exports car_<ID>.tga and car_spec_<ID>.tga. Auto-install on generate or copy anytime with one button.

Dual-folder install

Windows iRacing installs may use nested paths (stockcars2\arcaford25) or legacy flat folders (stockcars2 arcaford25). The app writes to both.

iRacing 3D preview

Installs your paint and launches the iRacing UI — open Paint Shop or Car Model to spin the real in-sim 3D viewer.

Demo mode

Works without API keys using a procedural fallback livery so you can try the full pipeline locally.

Quick start

git clone https://github.com/FocusTheRage/iracing-ai-paint-generator.git
cd iracing-ai-paint-generator
python -m pip install -r requirements.txt
copy .env.example .env
python app.py

Open http://127.0.0.1:7860 after launching. Add XAI_API_KEY to .env for cloud generation, or use demo mode without keys.

Using the app

  1. Select your car and enter your iRacing Customer ID (used for filenames only).
  2. Describe the livery and click Generate Paint.
  3. Download the TGAs, enable Auto-install to iRacing folder, or click Copy to iRacing Folder.
  4. Optional: Preview in iRacing 3D Viewer installs the paint and opens the iRacing UI.

Copy to iRacing

Paints are written under %USERPROFILE%\Documents\iRacing\paint\. The status panel shows every folder copied to, with checkmarks on each file. Stale .mip cache files are removed automatically; restart iRacing if an old design still appears.
